## 📑 슬라이드별 구성 및 스크립트

### [Slide 1] 표지
- **헤드라인**: 국가 재난·응급 알림 포털, **SafeSync**
- **서브텍스트**: 재난 상황에서도 절대 멈추지 않는 멀티 클라우드 DR 아키텍처
- **시각자료**: SafeSync 로고 + 다크 톤의 디지털 연결망 배경
- **🎤 스크립트**: "안녕하세요, 재난 상황에서도 절대 멈추지 않는 알림 시스템 'SafeSync'를 개발한 ○○조입니다. 발표 시작하겠습니다."

### [Slide 2] 기획 배경 (Problem)
- **헤드라인**: "재난 포털이 재난으로 접속 불가능해진다면?"
- **내용**: 단일 데이터센터 장애 시 대국민 정보 공백 발생 위험성. (예: 과거 메신저 서비스 데이터센터 화재 등)
- **🎤 스크립트**: "과거 데이터센터 화재로 온 국민의 통신망이 마비된 적이 있습니다. 만약 국가 재난을 알려야 하는 포털 사이트가 재난으로 인해 다운된다면 어떻게 될까요? 저희는 이 문제에 집중했습니다."

### [Slide 3] 솔루션 (Solution)
- **헤드라인**: Multi-Cloud Disaster Recovery
- **내용**: NCP(Primary)와 AWS(Secondary)를 융합한 이중화 구성 제안.
- **🎤 스크립트**: "SafeSync는 NCP를 메인으로, AWS를 재해복구(DR) 용도로 사용하는 멀티 클라우드 아키텍처를 도입하여 데이터 유실 없는 무중단 서비스를 실현했습니다."

### [Slide 4] 전체 시스템 아키텍처 (Core)
- **헤드라인**: SafeSync 시스템 아키텍처
- **시각자료**: *종합보고서 §3의 아키텍처 다이어그램 예쁘게 다시 그리기*
- **핵심 표기**: NCP ↔ AWS 간의 DB Replication, Route 53 분기 처리
- **🎤 스크립트**: "저희 시스템의 핵심 구조입니다. 평소에는 NCP를 통해 서비스가 제공되며, 내부적으로는 AWS RDS로 데이터가 실시간 복제됩니다. 앞단에는 Route 53이 Health Check를 통해 상태를 모니터링합니다."

### [Slide 5] 기술 스택 (Tech Stack)
- **헤드라인**: 적용된 클라우드 및 개발 기술
- **내용**:
  - Cloud: NCP (VPC, LB, ASG, Cloud DB), AWS (EC2, ALB, RDS, Route 53)
  - Backend/AI: FastAPI, MySQL, **CLOVA Studio (HCX-003)**, Docker
  - Frontend: Vanilla JS, CSS3 (Glassmorphism)
- **🎤 스크립트**: "도입된 기술 스택입니다. 특히 양쪽 클라우드에 동일한 환경을 배포하기 위해 Docker를 적극 활용했고, 상황에 맞는 대피 요령을 위해 네이버의 CLOVA Studio AI를 연동했습니다."

### [Slide 6] 기능 소개 1 - 직관적 UI & 대피소 현황
- **헤드라인**: 실시간 모니터링 및 대피소 정보
- **시각자료**: SafeSync 메인 대시보드 화면 + 대피소 수용률 화면 캡처
- **🎤 스크립트**: "사용자 화면입니다. 실시간 재난 위치를 지도로 확인하고, 근처 대피소의 현재 수용률을 바(bar) 형태로 직관적으로 파악할 수 있습니다."

### [Slide 7] 기능 소개 2 - CLOVA Studio AI 연동
- **헤드라인**: AI 기반 상황 맞춤형 행동 요령 생성
- **시각자료**: AI 채팅 입력창 + 결과 출력 화면 캡처
- **🎤 스크립트**: "단순히 재난을 알리는 것을 넘어, CLOVA Studio HCX-003 모델과 연동하여 현재 사용자가 처한 상황을 텍스트로 입력하면 가장 적합한 5단계 행동 요령을 즉시 생성해 줍니다."

### [Slide 8] 🔴 라이브 시연: DR Failover 테스트
- **헤드라인**: 재해 발생 시나리오 라이브 시연
- **시각자료**: (영상 또는 화면 전환을 위한 안내 슬라이드) "NCP 장애 발생 → AWS 자동 전환"
- **🎤 스크립트**:
  *(발표자가 화면을 관리자 콘솔 또는 실제 듀얼 웹 화면으로 전환)*
  "지금부터 저희가 구축한 DR 시스템이 실제로 동작하는지 시연해 보겠습니다. 현재는 NCP에서 정상 서비스 중입니다. 여기서 제가 고의로 NCP 서버에 장애를 발생시켜 보겠습니다. *(팀장님이 장애 버튼 클릭)*.
  Route 53이 장애를 감지하고 트래픽을 AWS로 전환합니다. 서비스 중단 없이 동일한 데이터를 AWS에서 확인하실 수 있습니다."

### [Slide 9] 기대 효과
- **헤드라인**: SafeSync가 가져올 가치
- **내용**:
  1. 무중단 대국민 정보 제공
  2. 데이터 유실(RPO) 제로에 가까운 안정성
  3. 특정 클라우드 벤더 종속성 탈피 (Multi-Cloud)
- **🎤 스크립트**: "SafeSync는 데이터 유실을 최소화하고, 클라우드 사업자 종속성에서 벗어나 안정적인 대국민 서비스를 제공하는 최적의 모델입니다."

## 💡 PPT 제작 팁 (팀원 C를 위한)
1. **아키텍처 그림이 가장 중요합니다.** 심사위원이 한눈에 들어오도록 깔끔한 아이콘(AWS, NCP 공식 아이콘)을 사용하여 그려주세요.
2. 캡처 이미지는 웹 브라우저 전체보다는 **주요 UI 요소만 크롭(Crop)**하여 넣는 것이 가독성이 좋습니다.
3. 8번 슬라이드는 **실제 시연(라이브 데모)**으로 넘어가는 브릿지 슬라이드입니다. 애니메이션이나 사이렌 아이콘 등으로 시선을 집중시켜 주세요.
